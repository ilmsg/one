<ul>
<li><strong>Two pointers:</strong> เทคนิคที่ใช้ตัวชี้ (pointer) สองตัวเคลื่อนที่ผ่านอาร์เรย์เพื่อแก้ปัญหาต่างๆ เช่น การหาคู่ที่รวมกันได้ค่าที่ต้องการ หรือการตรวจสอบพาลินโดรม</li>
<li><strong>Sliding window:</strong> เทคนิคที่ใช้ "หน้าต่าง" ขนาดคงที่หรือแปรผันเลื่อนไปตามอาร์เรย์เพื่อประมวลผลข้อมูลย่อยๆ มักใช้ในการหาค่าสูงสุด/ต่ำสุดในหน้าต่าง หรือหาลำดับย่อยที่ตรงตามเงื่อนไข</li>
<li><strong>Sorting:</strong> การจัดเรียงข้อมูลในอาร์เรย์ตามลำดับที่กำหนด (เช่น น้อยไปมาก หรือ มากไปน้อย) เป็นพื้นฐานสำคัญสำหรับอัลกอริทึมอื่นๆ</li>
</ul>
</li>
</ul>
</li>
<li>
<p><strong>Linked Lists (ลิงก์ลิสต์):</strong></p>
<ul>
<li><strong>คำจำกัดความ:</strong> เป็นลำดับขององค์ประกอบ (เรียกว่า โหนด - node) ที่แต่ละโหนดจะมีการอ้างอิง (pointer) ชี้ไปยังโหนดถัดไป (หรือโหนดก่อนหน้าในกรณีของ Doubly Linked List) ข้อมูลไม่จำเป็นต้องเก็บในหน่วยความจำที่อยู่ติดกัน</li>
<li><strong>การดำเนินการที่เกี่ยวข้อง:</strong>
<ul>
<li><strong>Reverse a linked list:</strong> การกลับลำดับของโหนดในลิงก์ลิสต์</li>
<li><strong>Detect a cycle:</strong> การตรวจสอบว่ามีโหนดใดในลิงก์ลิสต์ที่ชี้กลับไปยังโหนดก่อนหน้า ทำให้เกิดเป็นวงจรหรือไม่</li>
<li><strong>Merge lists:</strong> การรวมลิงก์ลิสต์สองรายการ (หรือมากกว่า) เข้าด้วยกัน โดยมักจะเรียงลำดับข้อมูลไปด้วย</li>
</ul>
</li>
</ul>
</li>
<li>
<p><strong>Stacks &#x26; Queues (สแต็กและคิว):</strong></p>
<ul>
<li><strong>คำจำกัดความ:</strong> เป็นโครงสร้างข้อมูลเชิงเส้นที่มีข้อจำกัดในการเพิ่มและลบข้อมูล
<ul>
<li><strong>Stack (สแต็ก):</strong> ทำงานแบบ LIFO (Last-In, First-Out) คือ ข้อมูลที่เข้ามาก่อนจะออกทีหลังสุด เหมือนกองจาน</li>
<li><strong>Queue (คิว):</strong> ทำงานแบบ FIFO (First-In, First-Out) คือ ข้อมูลที่เข้ามาก่อนจะออกก่อน เหมือนแถวรอคิว</li>
</ul>
</li>
<li><strong>อัลกอริทึม/การประยุกต์ใช้:</strong>
<ul>
<li><strong>Evaluate RPN (Reverse Polish Notation):</strong> การคำนวณนิพจน์ทางคณิตศาสตร์ที่เขียนในรูปแบบ RPN โดยใช้สแต็ก</li>
<li><strong>Parentheses validation:</strong> การตรวจสอบว่าวงเล็บในนิพจน์เปิดและปิดถูกต้องตามลำดับหรือไม่ โดยใช้สแต็ก</li>
<li><strong>Implement cache:</strong> การสร้างแคช (เช่น LRU Cache - Least Recently Used) ซึ่งมักใช้คิวและโครงสร้างข้อมูลอื่นร่วมด้วย</li>
</ul>
</li>
</ul>
</li>
<li>
<p><strong>Trees (ทรี):</strong></p>
<ul>
<li><strong>คำจำกัดความ:</strong> เป็นโครงสร้างข้อมูลแบบลำดับชั้น (hierarchical) ที่ประกอบด้วยโหนด (node) ที่มีความสัมพันธ์แบบพ่อ-ลูก (parent-child) โหนดบนสุดเรียกว่า ราก (root) และโหนดที่ไม่มีลูกเรียกว่า ใบ (leaf)</li>
<li><strong>ประเภทของทรี:</strong>
<ul>
<li><strong>Binary tree:</strong> ทรีที่แต่ละโหนดมีลูกได้ไม่เกินสองโหนด (ซ้ายและขวา)</li>
<li><strong>Binary search tree (BST):</strong> ทรีแบบไบนารีที่มีคุณสมบัติเพิ่มเติมคือ ค่าในโหนดลูกซ้ายทั้งหมดจะน้อยกว่าค่าในโหนดพ่อ และค่าในโหนดลูกขวาทั้งหมดจะมากกว่าค่าในโหนดพ่อ ทำให้ค้นหาข้อมูลได้เร็ว</li>
<li><strong>Trie (Prefix Tree):</strong> ทรีชนิดพิเศษที่ใช้เก็บและค้นหาสตริงตามคำนำหน้า (prefix) ได้อย่างมีประสิทธิภาพ</li>
</ul>
</li>
<li><strong>อัลกอริทึมที่พบบ่อย:</strong>
<ul>
<li><strong>Tree traversal:</strong> การเยี่ยมชมโหนดทุกโหนดในทรีตามลำดับที่กำหนด (เช่น In-order, Pre-order, Post-order, Level-order)</li>
<li><strong>Lowest common ancestor (LCA):</strong> การหาโหนดบรรพบุรุษร่วมที่อยู่ต่ำที่สุด (ใกล้โหนดลูกที่สุด) ของโหนดสองโหนดที่กำหนดในทรี</li>
<li><strong>Diameter of a tree:</strong> การหาระยะทางที่ยาวที่สุดระหว่างโหนดสองโหนดใดๆ ในทรี</li>
</ul>
</li>
</ul>
</li>
<li>
<p><strong>Searching (การค้นหา):</strong></p>
<ul>
<li><strong>คำจำกัดความ:</strong> เป็นกระบวนการในการหาตำแหน่งหรือการมีอยู่ของข้อมูลที่ต้องการในชุดข้อมูล</li>
<li><strong>อัลกอริทึมการค้นหา:</strong>
<ul>
<li><strong>Binary search:</strong> อัลกอริทึมการค้นหาที่มีประสิทธิภาพสูงสำหรับข้อมูลที่ <em>เรียงลำดับแล้ว</em> โดยแบ่งครึ่งช่วงการค้นหาไปเรื่อยๆ</li>
<li><strong>Depth-first search (DFS):</strong> การค้นหาในแนวลึก โดยจะสำรวจไปตามกิ่งก้านสาขาให้ลึกที่สุดก่อน แล้วจึงย้อนกลับมาสำรวจกิ่งอื่น มักใช้กับทรีและกราฟ</li>
<li><strong>Breadth-first search (BFS):</strong> การค้นหาในแนวกว้าง โดยจะสำรวจโหนดในระดับเดียวกันทั้งหมดก่อน แล้วจึงลงไปสำรวจระดับถัดไป มักใช้กับทรีและกราฟเพื่อหาระยะทางที่สั้นที่สุด</li>
<li><strong>A* w/ heuristics:</strong> อัลกอริทึมการค้นหาแบบมีข้อมูลนำทาง (informed search) ที่ใช้ฟังก์ชันฮิวริสติก (heuristic function) เพื่อประเมินค่าใช้จ่ายในการไปถึงเป้าหมาย ทำให้ค้นหาเส้นทางที่ดีที่สุดได้อย่างมีประสิทธิภาพ มักใช้ในการหาเส้นทางในกราฟ</li>
</ul>
</li>
</ul>
</li>
<li>
<p><strong>Sorting (การเรียงลำดับ):</strong></p>
<ul>
<li><strong>คำจำกัดความ:</strong> เป็นกระบวนการจัดเรียงองค์ประกอบในชุดข้อมูลตามลำดับที่กำหนด (เช่น จากน้อยไปมาก หรือตามลำดับตัวอักษร)</li>
<li><strong>อัลกอริทึมการเรียงลำดับ:</strong>
<ul>
<li><strong>Quick sort:</strong> อัลกอริทึมแบบ Divide and Conquer ที่มีความเร็วเฉลี่ยสูง เลือก pivot แล้วแบ่งข้อมูลเป็นส่วนที่น้อยกว่าและมากกว่า pivot แล้วทำซ้ำ</li>
<li><strong>Merge sort:</strong> อัลกอริทึมแบบ Divide and Conquer ที่มีประสิทธิภาพคงที่ แบ่งข้อมูลเป็นส่วนย่อยๆ เรียงลำดับส่วนย่อย แล้วผสาน (merge) กลับเข้าด้วยกัน</li>
<li><strong>Heap sort:</strong> อัลกอริทึมที่ใช้โครงสร้างข้อมูลแบบ Heap (โดยเฉพาะ Binary Heap) ในการเรียงลำดับ</li>
<li><strong>Insertion sort:</strong> อัลกอริทึมอย่างง่ายที่ทำงานโดยค่อยๆ สร้างส่วนที่เรียงลำดับแล้วทีละองค์ประกอบ เหมาะสำหรับข้อมูลเกือบเรียงลำดับอยู่แล้วหรือชุดข้อมูลขนาดเล็ก</li>
<li><strong>Topological sort:</strong> การเรียงลำดับสำหรับ Directed Acyclic Graph (DAG) โดยจัดเรียงโหนดตามลำดับที่โหนดหนึ่งต้องมาก่อนอีกโหนดหนึ่งตามทิศทางของเส้นเชื่อม ใช้ในการจัดการลำดับงานที่ต้องทำก่อนหลัง</li>
</ul>
</li>
</ul>
</li>
</ol>
<p>เอกสารสรุปนี้เป็นภาพรวมของหัวข้อสำคัญที่ควรเตรียมตัวสำหรับการสัมภาษณ์งานด้านเทคโนโลยี การทำความเข้าใจแนวคิดพื้นฐาน การทำงาน และความซับซ้อนทางเวลาและพื้นที่ (Time and Space Complexity) ของแต่ละโครงสร้างข้อมูลและอัลกอริทึมเหล่านี้เป็นสิ่งสำคัญอย่างยิ่ง</p>
