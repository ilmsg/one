<ul>
<li><strong>KPI Definition:</strong> อธิบายว่า KPI คือ "การแสดงออกที่สามารถวัดผลได้ สำหรับการบรรลุผลลัพธ์ที่ต้องการในขอบเขตที่เกี่ยวข้องกับกิจกรรมของหน่วยงาน" พูดง่ายๆ คือ เป็นค่าที่วัดว่าองค์กรบรรลุเป้าหมายสำคัญได้ดีเพียงใด</li>
<li><strong>SMART Objectives:</strong> เน้นว่าเป้าหมายที่ดีควรเป็นแบบ SMART ซึ่งย่อมาจาก Specific (เฉพาะเจาะจง), Measurable (วัดผลได้), Achievable (บรรลุได้), Relevant (เกี่ยวข้อง), และ Time-bound (มีขอบเขตเวลา) ตัวอย่างที่ให้คือ "เพิ่มฐานลูกค้าเพื่อให้ได้ส่วนแบ่งตลาด 20% ภายในสิ้นปีงบประมาณภายใต้การนำของผู้อำนวยการฝ่ายขาย" โดยมีองค์ประกอบคือ Objective (เป้าหมาย), KPI (% ส่วนแบ่งตลาด), Target (เป้าหมายที่ 20%), Timeframe (ภายในสิ้นปีงบประมาณ), และ Responsible (ผู้รับผิดชอบคือ ผู้อำนวยการฝ่ายขาย)</li>
<li><strong>KPI naming standards:</strong> แนะนำว่าชื่อ KPI ควรสื่อความหมายชัดเจน และมักจะขึ้นต้นด้วยสัญลักษณ์ เช่น $, #, % เพื่อบอกประเภทของค่าที่วัด (เช่น มูลค่า, จำนวน, เปอร์เซ็นต์)</li>
</ul>
</li>
<li>
<p><strong>Value added by KPIs (คุณค่าที่ได้จาก KPIs):</strong></p>
<ul>
<li><strong>Clarity (ความชัดเจน):</strong> ช่วยให้เห็นภาพรวมของกลยุทธ์องค์กรได้ชัดเจนขึ้น</li>
<li><strong>Focus (การมุ่งเน้น):</strong> ช่วยให้องค์กรมุ่งเน้นไปที่สิ่งที่สำคัญและต้องการความสนใจ</li>
<li><strong>Improvement (การปรับปรุง):</strong> ช่วยติดตามความคืบหน้าไปสู่เป้าหมายที่ต้องการ และชี้จุดที่ต้องปรับปรุง</li>
</ul>
</li>
<li>
<p><strong>Metrics - KPIs - KRIs - Analytics (การวัดผล - ตัวชี้วัดประสิทธิภาพหลัก - ตัวชี้วัดความเสี่ยงหลัก - การวิเคราะห์):</strong></p>
<ul>
<li>แสดงความสัมพันธ์ว่า <strong>Metrics (การวัดผล)</strong> เป็นพื้นฐาน จากนั้นนำไปสู่ <strong>KPIs (ตัวชี้วัดประสิทธิภาพหลัก)</strong> เพื่อประเมินผลการดำเนินงาน และ <strong>KRIs (Key Risk Indicators - ตัวชี้วัดความเสี่ยงหลัก)</strong> เพื่อประเมินความเสี่ยง ทั้งหมดนี้ถูกนำไปใช้ใน <strong>Analytics (การวิเคราะห์)</strong> เพื่อช่วยในการตัดสินใจ (Decision Making)</li>
</ul>
</li>
<li>
<p><strong>KPI selection criteria (เกณฑ์การเลือก KPI):</strong></p>
<ul>
<li><strong>Relevant (เกี่ยวข้อง):</strong> ต้องสอดคล้องกับกลยุทธ์ขององค์กร และมีความสำคัญต่อขอบเขตงานนั้นๆ</li>
<li><strong>Clearly defined (กำหนดชัดเจน):</strong> ควรระบุความหมายชัดเจน เข้าใจง่าย และหลีกเลี่ยงศัพท์เฉพาะทางเทคนิคที่ไม่จำเป็น</li>
<li><strong>Balanced (สมดุล):</strong> ควรพิจารณาความสมดุลระหว่าง คุณภาพ/ปริมาณ, ประสิทธิภาพ/ประสิทธิผล, ความเป็นอัตนัย/ความเป็นปรนัย</li>
</ul>
</li>
<li>
<p><strong>KPI selection workshop input (ข้อมูลนำเข้าสำหรับเวิร์กชอปการเลือก KPI):</strong></p>
<ul>
<li>ชี้ให้เห็นแหล่งข้อมูลที่ใช้ในการกำหนด KPI ซึ่งแบ่งเป็น:
<ul>
<li><strong>Primary sources (แหล่งข้อมูลปฐมภูมิ):</strong>
<ul>
<li><em>Internal (ภายใน):</em> พนักงานหน้างาน (Front-line employees), ผู้จัดการ (Managers), คณะกรรมการ (Board)</li>
<li><em>External (ภายนอก):</em> ซัพพลายเออร์ (Suppliers), ลูกค้า (Customers), คำแนะนำจากผู้เชี่ยวชาญ (Expert advice)</li>
</ul>
</li>
<li><strong>Secondary sources (แหล่งข้อมูลทุติยภูมิ):</strong> แผนพัฒนากลยุทธ์ (Strategic development plans), รายงานประจำปี (Annual reports), รายงานการปฏิบัติงาน (Operational reports), รายงานจากคู่แข่ง (Competitor reports), รายงานประจำปีขององค์กรอื่น ๆ (Annual reports of other organizations)</li>
</ul>
</li>
</ul>
</li>
<li>
<p><strong>Data Visualization (การแสดงข้อมูลด้วยภาพ):</strong></p>
<ul>
<li><strong>Dos (สิ่งที่ควรทำ):</strong> แนะนำให้ใช้กราฟแท่ง (Bar charts), กราฟเส้น (Line graph), กราฟหัวข้อย่อย (Bullet graph), Sparklines, กราฟหลายอันเล็กๆ (Small multiples), กราฟน้ำตก (Waterfall chart) ซึ่งมักจะสื่อความหมายได้ชัดเจนและเปรียบเทียบได้ง่าย</li>
<li><strong>Don'ts (สิ่งที่ไม่ควรทำ):</strong> แนะนำให้หลีกเลี่ยงกราฟวงกลม (Pie charts - โดยเฉพาะเมื่อมีหลายส่วน), กราฟ 3 มิติ (3D graphs - ทำให้บิดเบือนการรับรู้), พื้นหลังสีเข้ม (Dark backgrounds - อ่านยาก), แกนที่ไม่เริ่มจากศูนย์ (Non-zero baseline - สำหรับกราฟแท่ง ทำให้การเปรียบเทียบสัดส่วนผิดเพี้ยน), กราฟที่รกเกินไป (Overcrowded), และการใช้เส้นตารางมากเกินไป (Gridlines)</li>
</ul>
</li>
<li>
<p><strong>KPI selection technique: Value flow analysis (เทคนิคการเลือก KPI: การวิเคราะห์กระแสคุณค่า):</strong></p>
<ul>
<li>แสดงตัวอย่างการใช้เทคนิคนี้เพื่อปรับปรุงทักษะอย่างต่อเนื่องผ่านประสบการณ์การเรียนรู้ โดยแบ่งเป็น Input (ปัจจัยนำเข้า), Process (กระบวนการ), Output (ผลผลิต), และ Outcome (ผลลัพธ์)
<ul>
<li><strong>Input:</strong> งบประมาณการฝึกอบรม ($), เจ้าหน้าที่สนับสนุนการฝึกอบรม (#)</li>
<li><strong>Process:</strong> ชั่วโมงการฝึกอบรมต่อพนักงาน (#), หลักสูตรฝึกอบรมที่จัด (#)</li>
<li><strong>Output:</strong> พนักงานที่ได้รับการฝึกอบรม (%), ความพึงพอใจของผู้เข้าร่วมต่อประสบการณ์การฝึกอบรม (%)</li>
<li><strong>Outcome:</strong> พนักงานมีระดับความสามารถตามที่ต้องการ (%), ระดับทักษะ (#)</li>
</ul>
</li>
</ul>
</li>
<li>
<p><strong>KPI documentation (การจัดทำเอกสาร KPI):</strong></p>
<ul>
<li>แสดงตัวอย่างการจัดทำเอกสารสำหรับ KPI "อัตราการครองเตียงในโรงพยาบาล (%)" (Hospital bed occupancy rate) ประกอบด้วย:
<ul>
<li><strong>Name (ชื่อ):</strong> % Hospital bed occupancy rate</li>
<li><strong>Definition (คำจำกัดความ):</strong> วัดเปอร์เซ็นต์ของเตียงในโรงพยาบาลที่มีผู้ป่วยครอง จากจำนวนเตียงทั้งหมดในโรงพยาบาล</li>
<li><strong>Calculation (การคำนวณ):</strong>
<ul>
<li><em>Subordinate measures used (ค่าที่ใช้วัดรอง):</em> A = # เตียงที่มีผู้ป่วยครอง, B = # เตียงทั้งหมดในโรงพยาบาล</li>
<li><em>Calculation formula (สูตรคำนวณ):</em> (A / B) * 100</li>
<li><em>Formula type (ประเภทสูตร):</em> Rate (อัตราส่วน)</li>
<li><em>Trend is good when (แนวโน้มที่ดีคือ):</em> Within range (อยู่ในช่วงที่กำหนด)</li>
</ul>
</li>
<li><strong>Target (เป้าหมาย):</strong> ตัวอย่างเกณฑ์เป้าหมาย: สีแดง: &#x3C; 80% ; > 95%, สีเหลือง: 80-85% ; 90-95%, สีเขียว: 85-90% (แสดงว่าสูงหรือต่ำเกินไปอาจไม่ดี ต้องอยู่ในช่วงที่เหมาะสม)</li>
</ul>
</li>
</ul>
</li>
<li>
<p><strong>Functional Areas (ขอบเขตงานตามหน้าที่):</strong></p>
<ul>
<li>แสดงตัวอย่าง KPIs ที่ใช้กันทั่วไปในฝ่ายงานต่างๆ ขององค์กร เช่น บัญชี (Accounting), การปฏิบัติตามกฎระเบียบและความเสี่ยง (Compliance and Risk), บริการลูกค้า (Customer Service), การเงิน (Finance), อาชีวอนามัย ความปลอดภัย และสิ่งแวดล้อม (HSSE), ทรัพยากรบุคคล (Human Resources), เทคโนโลยีสารสนเทศ (Information Technology), การตลาดและการสื่อสาร (Marketing &#x26; Communication), การจัดซื้อและการกระจายสินค้า (Procurement &#x26; Distribution), การผลิต (Production), การจัดการโครงการ (Project Management), การจัดการคุณภาพ (Quality Management), การวิจัยและพัฒนา (Research &#x26; Development), และการขาย (Sales)</li>
</ul>
</li>
<li>
<p><strong>Industries (อุตสาหกรรม):</strong></p>
<ul>
<li>แสดงตัวอย่าง KPIs ที่ใช้ในอุตสาหกรรมต่างๆ เช่น ศูนย์บริการทางโทรศัพท์ (Call Center), ศุลกากร (Customs), การศึกษาและการฝึกอบรม (Education &#x26; Training), สถาบันการเงิน (Financial Institutions), รัฐบาล (Government - State/Federal), การดูแลสุขภาพ (Healthcare), การบริการและการท่องเที่ยว (Hospitality &#x26; Tourism), การดำเนินงานโครงสร้างพื้นฐาน (Infrastructure Operations), อสังหาริมทรัพย์ (Real Estate/Property), ทรัพยากร (Resources), ค้าปลีก (Retail), โทรคมนาคม (Telecommunications), การขนส่ง (Transportation), และสาธารณูปโภค (Utilities)</li>
</ul>
</li>
</ol>
<p>โดยรวมแล้ว อินโฟกราฟิกนี้เป็นคู่มือที่ยอดเยี่ยมสำหรับผู้ที่ต้องการทำความเข้าใจว่า KPI คืออะไร ทำไมจึงมีความสำคัญ จะเลือก KPI ที่ดีได้อย่างไร จะวัดผลและนำเสนอข้อมูลอย่างไร รวมถึงตัวอย่าง KPI ที่ใช้ในฝ่ายงานและอุตสาหกรรมต่างๆ ครับ</p>
